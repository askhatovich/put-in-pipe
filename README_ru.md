# Put-In-Pipe

Сервер потоковой передачи файлов с сохранением приватности. Сервер выступает посредником между отправителем и несколькими получателями, передавая файлы по частям (чанками) без сохранения полного файла на диск. Клиенты никогда не соединяются напрямую друг с другом -- IP-адреса участников не раскрываются.

## Возможности

- **Сервер как реле** -- клиенты общаются только через сервер, без прямых соединений
- **Потоковая передача чанками** -- размер файла не ограничен дисковым пространством или памятью сервера
- **Несколько получателей** в одной сессии (настраиваемый лимит)
- **Защита капчей** от автоматизированных злоупотреблений
- **Сквозное шифрование** -- XChaCha20-Poly1305 через libsodium; ключ передаётся в якоре URL (`#`), который никогда не отправляется на сервер
- **REST API + WebSocket** для событий передачи в реальном времени
- **Встроенный веб-интерфейс** -- Svelte 5 SPA, собранный в один HTML-файл и встроенный в бинарник сервера
- **Настройка** через INI-файл

## Сборка

### Только бэкенд

Требуется компилятор с поддержкой C++20 и CMake 3.16+. Если фронтенд ещё не собран, его нужно собрать предварительно (см. ниже).

```sh
cmake -B build src
cmake --build build
```

### Полная сборка (фронтенд + бэкенд)

Дополнительно к C++ нужны Node.js 18+ и npm.

```sh
# 1. Сборка фронтенда
cd web
npm install
npm run build    # создаёт web/dist/index.html (одностраничное приложение)
./embed.sh       # генерирует src/generated_index_html.h
cd ..

# 2. Сборка бэкенда (с встроенным фронтендом)
cmake -B build src
cmake --build build
```

После этого `GET /` отдаёт полноценное веб-приложение — отдельный статик-сервер не нужен.

### Разработка фронтенда

Фронтенд находится в `web/` и представляет собой стандартный проект Vite + Svelte 5.

```sh
cd web
npm install
npm run dev      # Vite dev-сервер с hot reload (http://localhost:5173)
npm run build    # продакшн-сборка → web/dist/index.html
```

При разработке запускайте `npm run dev` и настройте браузер для проксирования API-запросов к бэкенду (`http://localhost:2233`), или запускайте бэкенд параллельно и открывайте URL dev-сервера Vite.

**Стек:**
- [Svelte 5](https://svelte.dev/) -- реактивный UI-фреймворк
- [Vite](https://vite.dev/) + [vite-plugin-singlefile](https://github.com/nickreese/vite-plugin-singlefile) -- сборщик, инлайнит все ассеты в один HTML
- [libsodium-wrappers-sumo](https://github.com/nickvergessen/libsodium.js) -- XChaCha20-Poly1305 шифрование в браузере

**Структура проекта:**
```
web/
  src/
    App.svelte              -- корневой компонент, машина состояний
    lib/
      api.js                -- клиент REST API
      ws.js                 -- WebSocket-клиент с диспетчером событий
      crypto.js             -- обёртка libsodium (шифрование/расшифровка)
      i18n.js               -- переводы en/ru, автоопределение языка
      url.js                -- парсинг и формирование якоря URL (#)
    components/
      EntryScreen.svelte    -- начальный экран (выбор отправки/получения)
      Captcha.svelte        -- отображение и ввод капчи
      SessionSender.svelte  -- интерфейс отправителя
      SessionReceiver.svelte-- интерфейс получателя
      TransferComplete.svelte-- экран результата передачи
      MemberList.svelte     -- панель участников сессии
      ProgressPanel.svelte  -- прогресс файла и статус буфера
      Footer.svelte         -- статистика сервера + копирайт
```

### Встраивание фронтенда

Скрипт `web/embed.sh` конвертирует собранный `web/dist/index.html` в C++ заголовочный файл (`src/generated_index_html.h`), содержащий HTML в виде массива байт. Бэкенд подключает этот заголовок и отдаёт его на `/`.

Файл генерируется автоматически и добавлен в `.gitignore` — его нужно пересоздавать после любого изменения фронтенда:

```sh
cd web && npm run build && ./embed.sh
```

## Запуск

```sh
# Показать справку
./build/put-in-pipe --help

# Сгенерировать конфигурационный файл по умолчанию
./build/put-in-pipe --generate-config ./config.ini

# Запустить сервер
./build/put-in-pipe --config ./config.ini
```

### Параметры командной строки

| Параметр | Описание |
|---|---|
| `-c, --config <путь>` | Путь к файлу конфигурации (по умолчанию: `/etc/put-in-pipe/config.ini`) |
| `-g, --generate-config <путь>` | Сгенерировать конфигурацию по умолчанию по указанному пути |
| `-h, --help` | Показать справку |

## Конфигурация

Все доступные параметры описаны в [`contrib/config.ini.example`](contrib/config.ini.example). Основные секции:

- **`[server]`** -- адрес и порт привязки
- **`[client]`** -- максимум клиентов, порог капчи, время жизни капчи, таймаут клиента
- **`[session]`** -- лимит сессий, максимальный размер чанка, размер очереди буфера, время жизни сессии, максимум получателей, длительность начальной заморозки

## Тестирование

```sh
cd build && ctest --output-on-failure
```
