# Put-In-Pipe

Сервер потоковой передачи файлов с сохранением приватности. Сервер выступает посредником между отправителем и несколькими получателями, передавая файлы по частям (чанкам) без сохранения полного файла на диск. Клиенты никогда не соединяются напрямую друг с другом -- IP-адреса участников не раскрываются.

## Возможности

- **Сервер как реле** -- клиенты общаются только через сервер, без прямых соединений
- **Потоковая передача чанками** -- размер файла не ограничен дисковым пространством или памятью сервера
- **Несколько получателей** в одной сессии (настраиваемый лимит)
- **Защита капчей** от автоматизированных злоупотреблений
- **Возможность E2E-шифрования** на стороне фронтенда (например, ChaCha20-Poly1305 с ключом в якоре URL, невидимым серверу)
- **REST API + WebSocket** для событий передачи в реальном времени
- **Настройка** через INI-файл

## Сборка

Требуется компилятор с поддержкой C++20 и CMake 3.16+.

```sh
cmake -B build src
cmake --build build
```

## Запуск

```sh
# Показать справку
./build/put-in-pipe --help

# Сгенерировать конфигурационный файл по умолчанию
./build/put-in-pipe --generate-config ./config.ini

# Запустить сервер
./build/put-in-pipe --config ./config.ini
```

### Параметры командной строки

| Параметр | Описание |
|---|---|
| `-c, --config <путь>` | Путь к файлу конфигурации (по умолчанию: `/etc/put-in-pipe/config.ini`) |
| `-g, --generate-config <путь>` | Сгенерировать конфигурацию по умолчанию по указанному пути |
| `-h, --help` | Показать справку |

## Конфигурация

Все доступные параметры описаны в [`contrib/config.ini.example`](contrib/config.ini.example). Основные секции:

- **`[server]`** -- адрес и порт привязки
- **`[client]`** -- максимум клиентов, порог капчи, время жизни капчи, таймаут клиента
- **`[session]`** -- лимит сессий, максимальный размер чанка, размер очереди буфера, время жизни сессии, максимум получателей, длительность начальной заморозки

## Тестирование

Тесты используют Google Test и собираются по умолчанию.

```sh
cd build && ctest --output-on-failure
```
